# 🎮 AI 游戏买手 · 产品设计文档 v1.0

## 1. 产品定位

### 一句话描述
一个“懂情绪、有品味、会主动推荐”的 AI 游戏买手，帮助玩家在信息过载的时代快速找到真正想玩的游戏。

### 核心特色
- **多智能体讨论机制**：三个不同视角的 Agent（硬核、艺术、省钱）先“内部讨论”，再由主持人汇总成一段有温度的朋友式推荐。
- **情绪/场景匹配**：不止基于类型，更基于用户当前的心情和场景。
- **跨平台数据**：整合 Steam、Epic、PSN、Switch 等平台信息，包括价格、折扣、评分。

### 目标用户
- **普通玩家**：面对海量游戏不知道玩什么
- **硬核玩家**：想发掘冷门佳作
- **纠结型玩家**：希望有人帮做决策

---

## 2. 用户交互流程

用户通过网页聊天界面与买手互动，典型对话流程如下：

1. **用户提问**
   - 例如：“推荐几款像《星露谷物语》的游戏”
   - 或：“今天心情不好，想找个解压的”

2. **系统触发多 Agent 并行思考**
   - 后台同时唤醒三个 Agent：硬核顾问、艺术顾问、省钱顾问
   - 每个 Agent 根据用户意图和自己的专业领域，从游戏库中检索并生成一段推荐理由

3. **主持人 Agent 汇总**
   - 收集三个 Agent 的输出，整合成一段连贯、自然的回复
   - 回复包含：各 Agent 的观点摘要 + 最终综合建议 + 相关游戏链接/按钮

4. **用户反馈**
   - 用户可以选择“喜欢”或“再推荐”，或继续追问

## 3. 技术架构设计

采用轻量级微服务架构。

```text
[前端] 网页聊天界面 (Next.js/React)
        ↓ (HTTP)
[后端] FastAPI/Next.js API
        ├── 意图识别模块 (调用 LLM)
        ├── 任务分发器
        ├── Agent 1: 硬核顾问 (调用 LLM + 游戏库 API)
        ├── Agent 2: 艺术顾问 (调用 LLM + 游戏库 API)
        ├── Agent 3: 省钱顾问 (调用 LLM + 价格 API)
        └── 主持人 Agent (调用 LLM，汇总生成回复)
[数据层] 游戏库 (初期可用 RAWG/IGDB API + 本地缓存)
[外部 API]
    - 大模型 API (OpenAI/Gemini/DeepSeek 等)
    - 游戏数据 API (RAWG, IGDB)
    - 价格 API (Steam 官方 API)
```

### 3.1 核心组件说明

| 组件 | 职责 | 技术实现提示 |
| :--- | :--- | :--- |
| 前端 | 提供聊天界面，展示回复 | React + Tailwind CSS |
| 意图识别 | 分析输入，提取关键信息 | 调用 LLM，设计 Prompt |
| 任务分发器 | 并行调用 Agent | asyncio 或 Promise.all |
| 硬核顾问 | 从机制、难度、深度角度推荐 | 调用 LLM + 游戏库检索 |
| 艺术顾问 | 从美术、音乐、叙事角度推荐 | 同上 |
| 省钱顾问 | 考虑价格、折扣、性价比 | 接入价格 API |
| 主持人 | 汇总观点，生成友好回复 | 调用 LLM，设计 Prompt |
| 游戏库 | 存储游戏元数据 | RAWG API + 本地缓存 |

## 4. Agent 提示词设计

### 4.1 意图识别模块
**输入**：用户消息
**输出**：结构化意图（JSON）

```text
你是一个游戏推荐助手的意图识别专家。请从用户消息中提取以下信息，并以 JSON 格式返回：
- game_name: 提到的游戏名（如果有）
- emotion: 情绪关键词（如“累”、“开心”、“无聊”等）
- scenario: 场景（如“一个人玩”、“情侣”、“短时间”等）
- preferences: 其他偏好（如“像素风”、“策略”、“剧情”等）
- price_range: 价格范围（如有）
- platform: platform 偏好（如有）

用户消息：{用户输入}
```

### 4.2 硬核顾问 Agent
**角色设定**：资深硬核玩家，关注游戏机制、操作深度、系统复杂度、耐玩性。

```text
你是一个硬核游戏顾问，只从游戏机制、难度、系统深度、可重玩性角度推荐游戏。
用户意图：{意图JSON}
游戏库数据：{相关游戏列表}

请根据用户意图，从硬核角度推荐 3 款游戏。说明其机制/深度为何适合用户，给出专业见解。
输出格式：一段自然语言，体现硬核感。
```

### 4.3 艺术顾问 Agent
**角色设定**：游戏艺术爱好者，关注美术风格、音乐、叙事、氛围营造。

```text
你是一个艺术型游戏顾问，只从美术风格、音乐、叙事、氛围角度推荐游戏。
用户意图：{意图JSON}
游戏库数据：{相关游戏列表}

请根据用户意图，从艺术角度推荐 3 款游戏。描述美术特点、音乐如何烘托情绪、故事是否动人。
输出格式：一段自然语言，有文艺气息。
```

### 4.4 省钱顾问 Agent
**角色设定**：精明玩家，关注价格、折扣、性价比，提醒避坑。

```text
你是一个省钱型游戏顾问，只从价格、折扣、性价比、是否值得现在买的角度推荐游戏。
用户意图：{意图JSON}
价格信息：{价格 API 返回}

请推荐 1-3 款游戏，强调当前价格、史低情况、是否值得入手。
输出格式：一段自然语言，带点“精明”的语气。
```

### 4.5 主持人 Agent
**角色设定**：总结者，将三个顾问的观点融合成一段连贯、友好的回复。

```text
你是一个游戏买手主持人，需要将三位顾问（硬核、艺术、省钱）的观点整合，以朋友的口吻推荐。
用户原话：{用户输入}
硬核顾问观点：{硬核输出}
艺术顾问观点：{艺术输出}
省钱顾问观点：{省钱输出}

结构建议：
1. 简短回应用户情绪或需求。
2. 分别引用三位顾问的精华。
3. 给出综合建议及小结。
4. 回复亲切，不啰嗦。
```

## 5. 游戏库设计（MVP 版）

初期采用 **RAWG API + 本地缓存**。

### 5.1 数据获取流程
- 搜索相关游戏：调用 RAWG 的 `/games` 接口。
- 提取元数据：名称、封面、评分、类型、标签、平台、简介。
- 本地缓存：避免重复请求。

### 5.2 数据增强
- 手动添加“情绪标签”（如“治愈”、“解压”）。
- 后期引入 SteamSpy 或 IGDB API。

---

## 6. MVP 实现步骤

- **第 1 周**：单人 Agent 原型（前端聊天框 + 意图识别 + 硬核顾问 + RAWG）。
- **第 2 周**：双 Agent（增加省钱顾问 + 价格 API）。
- **第 3 周**：三 Agent + 主持人（完整讨论机制）。
- **第 4 周**：优化与测试（Prompt 优化 + 记忆能力）。

---

## 7. “买手”特色体现
- **主动推送**（后期）：基于历史每周精选。
- **记忆能力**（后期）：记住玩过的游戏和喜好。
- **人格化**：具有名字和头像，像真实朋友。
- **透明讨论**：展示顾问间的“观点交锋”。
